# Step 02: Core Experience & Emotional Strategy

Define the core user interaction, emotional goals, and inspiration patterns.

<critical>Three phases in one step: Core Experience → Emotional Response → Inspiration.
Each phase builds on the previous. Present A/P/C after all three are drafted.</critical>

---

## Phase A: Core Experience Definition

<action>Identify the most important user interaction:

  **Core Experience Questions:**
  - What's the ONE thing users do most frequently?
  - What should be completely effortless?
  - If we nail one interaction, everything else follows — what is it?

  **Platform Requirements:**
  - Web, mobile, desktop, or multiple?
  - Touch-based or mouse/keyboard?
  - Offline functionality needed?
  - Device-specific capabilities to leverage?

  **Effortless Interactions:**
  - What should feel natural and require zero thought?
  - Where do competitors require extra steps we can eliminate?

  **Critical Success Moments:**
  - When does the user realize "this is better"?
  - What interaction, if failed, ruins the experience?

  Synthesize into 3-5 Experience Principles.</action>

## Phase B: Emotional Response

<action>Explore how the product should make users feel:

  **Primary emotional goals** — empowered? delighted? efficient? calm?

  **Emotional Journey:**
  - First discovery → core action → task completion → return visit
  - What if something goes wrong?

  **Micro-Emotions:** confidence/confusion, trust/skepticism,
  excitement/anxiety, accomplishment/frustration

  **Design Implications:** Connect each emotional goal to specific
  UX design approach (e.g., "trust → progressive disclosure + clear feedback")</action>

## Phase C: UX Pattern Analysis & Inspiration

<action>Learn from existing products:

  Ask user to name 2-3 apps their target users love.
  For each: What core problem does it solve elegantly?
  What interaction patterns could transfer?

  Extract:
  - **Transferable Patterns** — navigation, interaction, visual
  - **Anti-Patterns to Avoid** — what creates friction in competitors
  - **Design Inspiration Strategy** — what to adopt, adapt, or avoid</action>
